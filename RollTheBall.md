It was a little bit challenging for me at first as I didn't do anything in Unity before this course, but at the end, it was quite satisfying to have a fully playable game.

**Getting used to the Unity Editor**

Before I began the actual course tutorial for Roll-a-ball, I went through the Tutorial on Getting Started With Unity Editor. That helped me a lot in understanding the basics, like how to create a new project and set up the layout. Passing through the Scene view and flying around was the main problem for me at first. Figuring out how to even go through the scene view to see my game objects through different angles felt really weird, but I just needed practice to get the hang of the camera controls.

**Global vs. Local Coordinates (and falling enemies!)**

Later on in the main course tutorial, I ran into some really funny technical problems. One of the big ones was figuring out the difference between switching from Global to Local coordinates. I was having trouble moving the objects so they wouldn't just go right through the ground, hah.

Because of that confusion with the axes, I made a hilarious mistake with the enemy. I accidentally moved the enemy up instead of forward. So, every time the game entered play mode, the playable character got eliminated instantly because the enemy was just falling out of the sky right onto him!

**The 20-minute CamelCase mistake**

I also had a really frustrating problem when creating the player ball that would roll. I created the movement script, but I wrote it without using the proper CamelCase formatting. I was sitting there trying to find the error for 20 minutes and almost gave up on the whole thing, hah. But at the end of the day, it was definitely worth it bc now every time I encounter a scripting problem or an error in Unity, checking my spelling and CamelCase is the very first thing that pops up in my head. It was a hard lesson but a good one.

**Coding the Enemy Behavior**

Despite the bugs, what I think was the most interesting part of the project was the enemy creation. It was really cool figuring out how it was adding specifications for it, like setting up the speed variable in the script so it could chase the player. Learning how to code what happens to the player when they catch them and handling the collisions made it feel like an actual game. Even though I had zero experience and made some silly mistakes, making this first game actually work was a great experience.
